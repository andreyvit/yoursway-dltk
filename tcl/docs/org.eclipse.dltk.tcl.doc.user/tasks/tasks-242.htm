Browsing the refactoring history

  1. Select Refactor > History... from the refactoring menu to show the refactoring history dialog.
  2. The refactoring history dialog displays the entire refactoring history of your workspace. Drill down to an individual refactoring and select it to see more details about the performed refactoring in the details view below.
  3. This dialog may also be used to remove certain refactorings from the refactoring history. Select an entire year, month, week or day and press Remove to remove the selected refactorings from the refactoring history.
  4. In order to be able to better select refactorings from the refactoring history, there are two ways to group your refactorings:
    • Group by Project: Refactorings are grouped by project. Every refactoring is associated with the project that contains the element which served as input to the recorded refactoring.
      Note: There are certain instances of refactorings which cannot be uniquely associated with a project. These refactorings are associated with the entire workspace.
    • Group by Date: Refactorings are grouped by date. The refactoring history dialog presents the refactorings as a history, last recently executed refactorings first.
